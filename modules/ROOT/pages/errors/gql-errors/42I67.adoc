= 42I67

== Status description
error: syntax error or access rule violation - unsupported language feature. The query is parsable in `CYPHER { <<version>>2 }`, but it is run in `CYPHER { <<version>>1 }`. Consider changing the database default Cypher version using `ALTER DATABASE SET DEFAULT LANGUAGE` or prefix the query with `CYPHER { <<feat>>2 }`.

== Example scenario

For example, when using `LET` in a Cypher 5 query:

[source,cypher]
----
CYPHER 5
LET x = 1
RETURN x;
----

The query returns a chain of errors where GQLSTATUS 42I67 is the cause of GQLSTATUS xref:errors/gql-errors/42I06.adoc[42I06], which is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:
