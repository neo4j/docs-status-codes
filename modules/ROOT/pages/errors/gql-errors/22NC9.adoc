= 22NC9

== Status description
error: data exception - invalid element type constraints in graph type. A `{ <<entityType>>1 }` element type property `{ <<context>> }` constraint cannot be specified inline of a `{ <<entityType>>2 }` element type.

== Explanation
Property type and existence constraints must be defined as part of the property description, not as inline constraints on graph type elements.

== Example scenario
Try defining an existent constraint inline of a node element type as follows:

[source,cypher]
----
ALTER CURRENT GRAPH TYPE SET {
    (p:Person => {name :: STRING, age :: INT}) REQUIRE p.studId IS NOT NULL
};
----

The query returns a chain of errors where GQLSTATUS 22NC9 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
22NC9: data exception - invalid element type constraints in graph type. A node element type property existence constraint cannot be specified inline of a node element type. (line 2, column 48 (offset: 78))
"    (p:Person => {name :: STRING, age :: INT}) REQUIRE p.studId IS NOT NULL"
                                                ^
  42001: syntax error or access rule violation - invalid syntax
----

To fix this, change the query as follows:

[source, cypher]
----
ALTER CURRENT GRAPH TYPE SET {
    (p :Person => {name :: STRING, age :: INT, studId :: ANY NOT NULL })
};
----
