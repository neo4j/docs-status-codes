= 42I64

== Status description
error: syntax error or access rule violation - unsupported operation. `{ <<msg>> }` currently not supported after `+NEXT+` `{ <<context>> }`.

== Example scenario

For example, when aggregating in a `UNION` query after a `NEXT`:

[source,cypher]
----
UNWIND [1, 2] AS x
RETURN x

NEXT

RETURN COUNT(x)
UNION ALL
RETURN COUNT(x);
----

// The example above doesn't return an error:
// neo4j@neo4j> UNWIND [1, 2] AS x
//              RETURN x

//              NEXT

//              RETURN COUNT(x)
//              UNION ALL
//              RETURN COUNT(x);
// +----------+
// | COUNT(x) |
// +----------+
// | 2        |
// | 2        |
// +----------+

// 2 rows
// ready to start consuming query after 192 ms, results consumed after another 10 ms

The query returns a chain of errors where GQLSTATUS 42I64 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
42I64: syntax error or access rule violation - unsupported operation. Aggregations are currently not supported in `NEXT` when used in a `UNION`.
  42001: syntax error or access rule violation - invalid syntax
----
