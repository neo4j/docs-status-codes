= 42N7B

== Status description
error: syntax error or access rule violation - non-deterministic disjoint by expression. `'DISJOINT BY'` expressions must be deterministic.

== Example scenario
For example, when the `DISJOINT BY` expression is non-deterministic, such as `rand()`:

[source,cypher]
----
UNWIND [1, 2, 3] AS i
CALL (i) {
  MATCH (n {id: i})
  SET n.processed = true
} IN CONCURRENT TRANSACTIONS DISJOINT BY (rand());
----

The query returns a chain of errors where GQLSTATUS 42N7B is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
42N7B: syntax error or access rule violation - non-deterministic disjoint by expression. 'DISJOINT BY' expressions must be deterministic. (line 5, column 43 (offset: 120))
"} IN CONCURRENT TRANSACTIONS DISJOINT BY (rand())"
                                           ^
  42001: syntax error or access rule violation - invalid syntax
----

== Possible solutions

To resolve this issue, replace the non-deterministic expression with a deterministic expression over the row variables, for example the variable itself.

[source,cypher]
----
UNWIND [1, 2, 3] AS i
CALL (i) {
  MATCH (n {id: i})
  SET n.processed = true
} IN CONCURRENT TRANSACTIONS DISJOINT BY (i);
----
