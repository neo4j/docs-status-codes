= 42N25

== Status description
error: syntax error or access rule violation - missing `YIELD`. Procedure call inside a query does not support naming results implicitly. Use `YIELD` instead. Available output columns are: `{ <<variableList>> }`.

== Explanation
When a non-void procedure is called inside a query, the results must be explicitly named using a `YIELD` clause.
Without `YIELD`, the query fails.
The error message includes the available output columns to help construct the correct `YIELD` clause.

== Example scenario
Try calling the built-in `db.labels()` procedure inside a query without `YIELD`:

[source,cypher]
----
MATCH (n)
CALL db.labels()
RETURN *;
----

The query returns a chain of errors where GQLSTATUS 42N25 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error
[source, errror]
----
42N25: syntax error or access rule violation - missing YIELD. Procedure call inside a query does not support naming results implicitly. Use YIELD instead. Available output columns are: `label`. (line 2, column 1 (offset: 10))
"CALL db.labels()"
 ^
  42001: syntax error or access rule violation - invalid syntax
----

== Possible solutions

To resolve this issue, add a `YIELD` clause naming the output columns you need:

[source,cypher]
----
MATCH (n)
CALL db.labels() YIELD label
RETURN n, label;
----

To discover a procedure's output signature, use `SHOW PROCEDURES`:

[source,cypher]
----
neo4j@system> SHOW PROCEDURES YIELD name, signature WHERE name = 'db.labels';

| name        | signature                          |
+--------------------------------------------------+
| "db.labels" | "db.labels() :: (label :: STRING)" |
+--------------------------------------------------+
