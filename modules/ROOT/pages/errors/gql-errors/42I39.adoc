= 42I39

== Status description
error: syntax error or access rule violation - invalid use of shortest path function. Mixing the `{ <<fun>> }` function with path selectors, explicit match modes, or explicit path modes is not allowed.

== Example scenario
Try running the following query `REPEATABLE ELEMENTS` with the `shortestPath()` function:

[source,cypher]
----
MATCH REPEATABLE ELEMENTS p = shortestPath((:A)-[:R*0..10]->(:B))
WHERE any(x IN nodes(p) WHERE x:X)
RETURN p;
----

The query returns a chain of errors where GQLSTATUS 42I39 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
42I39: syntax error or access rule violation - invalid use of shortest path function. Mixing the shortestPath() function with path selectors, explicit match modes or explicit path modes is not allowed. (line 1, column 31 (offset: 30))
"MATCH REPEATABLE ELEMENTS p = shortestPath((:A)-[:R*0..10]->(:B))"
                               ^
  42001: syntax error or access rule violation - invalid syntax
----

== Possible solution
Replace `shortestPath()` with the `SHORTEST 1` path selector, and `allShortestPaths()` with the `SHORTEST ALL` path selector.

For example, the query above can be rewritten as:

[source,cypher]
----
MATCH REPEATABLE ELEMENTS p = SHORTEST 1 (:A)-[:R*0..10]->(:B)
  WHERE any(x IN nodes(p) WHERE x:X)
RETURN p;
----
