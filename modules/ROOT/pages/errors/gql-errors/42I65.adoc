= 42I65

== Status description
error: syntax error or access rule violation - invalid pattern character. An invalid character is used in the pattern. Verify that all characters are supported by `{ <<valueType>> }`.

== Example scenario

Try formatting using a year character on a time-only type:

[source,cypher]
----
RETURN format(time(), "yyyy") AS formatted;
----

The query returns a chain of errors where GQLSTATUS 42I65 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
42I65: syntax error or access rule violation - invalid pattern character. An invalid character is used in the pattern. Verify that all characters are supported by `ZONED TIME`.
  42001: syntax error or access rule violation - invalid syntax
----
