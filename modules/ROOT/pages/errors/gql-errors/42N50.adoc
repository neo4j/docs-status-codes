= 42N50


== Status description
error: syntax error or access rule violation - procedure return column not defined. The procedure return column `{ <<arg>> }` is not defined for this procedure. Available output columns are: `{ <<argList>> }`. Verify that the spelling is correct.

== Explanation
When calling a procedure with a `YIELD` clause, all yielded column names must match the procedure's output signature.
If a column name is misspelled or does not exist in the procedure's output, the query will fail.
The error message includes the available output columns to help identify the correct name.

== Example scenario
For example, when calling the built-in `dbms.components()` procedure and yielding a non-existent column:

.Returns a non-existent column `nonExistent`
[source,cypher]
----
CALL dbms.components() YIELD nonExistent
RETURN nonExistent;
----

The query returns a chain of errors where GQLSTATUS 42N50 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
42N50: syntax error or access rule violation - procedure return column not defined. The procedure return column 'nonExistent' is not defined for this procedure. Available output columns are: 'edition', 'name' and 'versions'. Verify that the spelling is correct. (line 1, column 30 (offset: 29))
"CALL dbms.components() YIELD nonExistent"
                              ^
  42001: syntax error or access rule violation - invalid syntax
----

== Possible solutions

To resolve this issue, replace the invalid column name with at least one of the available output columns listed in the error message:

.Returns all available columns
[source,cypher]
----
CALL dbms.components() YIELD name, versions, edition
RETURN name, versions, edition;
----

.Returns only the `name` column
[source,cypher]
----
CALL dbms.components() YIELD name
RETURN name;
----

To discover a procedure's output signature, use `SHOW PROCEDURES`:

[source,cypher]
----
neo4j@system> SHOW PROCEDURES YIELD name, signature WHERE name = 'dbms.components';
+----------------------------------------------------------------------------------------------------------+
| name              | signature                                                                            |
+----------------------------------------------------------------------------------------------------------+
| "dbms.components" | "dbms.components() :: (name :: STRING, versions :: LIST<STRING>, edition :: STRING)" |
+----------------------------------------------------------------------------------------------------------+

1 row
ready to start consuming query after 321 ms, results consumed after another 11 ms
----
