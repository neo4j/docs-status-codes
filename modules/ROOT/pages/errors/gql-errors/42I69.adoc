= 42I69

== Status description
error: syntax error or access rule violation - invalid search variable reference. `{ <<variable>> }` must reference a variable from the same `MATCH` statement.

== Example scenario

Try using a variable from a previous `MATCH` statement as variable reference in a `SEARCH` clause:

[source,cypher]
----
CYPHER 25
MATCH (movie:Movie)
MATCH (m:Movie {title:'Matrix, The'})
  SEARCH movie IN (
    VECTOR INDEX moviePlots
    FOR [1, 2, 3]
    LIMIT 5
  )
RETURN movie.title AS title;
----

The query returns a chain of errors where GQLSTATUS 42I69 is the cause of GQLSTATUS xref:errors/gql-errors/42001.adoc[42001]:

.GQLSTATUS error chain
[source, error]
----
42I69: syntax error or access rule violation - invalid search variable reference. `movie` must reference a variable from the same MATCH statement. (line 4, column 10 (offset: 77))
"  SEARCH movie IN ("
          ^
  42001: syntax error or access rule violation - invalid syntax
----
